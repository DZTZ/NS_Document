Welcome! This guide explains the Blueprint nodes to help you quickly integrate Unreal Engine replay functionality.

Start Recording (Start Recording)

This node is used to initiate the recording process for the current game session. You can specify a unique name for each recording to make it easier to locate and play back later.

Blueprint Node:

Input Parameters (Left Pins)

Parameter Name Data Type Description
Exec (Top) Exec **Trigger:** Starts the recording logic; the system begins capturing game data immediately.
Replay Name String **Required:** Specifies the unique filename (ID) for this recording. Use unique identifiers to avoid overwriting old files.

Output Parameters (Right Pins)

Parameter Name Data Type Description
Exec (Top) Exec Triggers immediately after the recording command is issued. Note: This only indicates the "Start" operation was executed.
Return Value Boolean **Success Flag:** Returns `True` if recording started successfully; `False` if it failed (e.g., due to permissions, disk space, or an active recording).

Stop Recording (Stop Recording)

This node is used to stop the current game recording. Once called, the system finalizes the data and saves the replay file.

Blueprint Node:

Input Parameters

Parameter Name Data Type Description
Exec (Top) Exec **Trigger:** Stops the recording logic.

Output Parameters

Parameter Name Data Type Description
Exec (Top) Exec Triggers after the stop command is issued.
Return Value Boolean **Success Flag:** Returns `True` if recording stopped successfully; `False` if no recording was currently active.

Get Replay List (Get Replay List)

This node is used to retrieve a list of all available replay files from the storage device.

Blueprint Node:

Output Parameters

Parameter Name Data Type Description
Out Replay List Array (Struct) **Core Data:** An array containing replay information. Each element includes the name, path, and other struct data.
Return Value Boolean **Success Flag:** Whether the list was successfully retrieved (returns True even if the list is empty).

Data Structure: Replay Info (via Break Node)

  • Replay Name (String): The unique name assigned during recording.
  • Replay Path (String): The physical storage path of the file on the disk.

Play Replay (Play Replay)

This node initiates the playback functionality. The system searches for a file matching the input name and switches to replay spectator mode.

Blueprint Node:

Input Parameters

Parameter Name Data Type Description
Exec Exec Triggers the loading and playback logic.
Replay Name String **Required:** The name of the replay file to play (e.g., myReplay_2025-12-06 19-56).

Output Parameters

Parameter Name Data Type Description
Exec Exec Triggers immediately after the play command is issued.
Return Value Boolean **Success Flag:** Returns True if the file was found and playback started; False if the file is missing or failed to load.

Delete Replay (Delete Replay)

Used to clean up unwanted recording data. Note: Deletion is permanent and irreversible.

Blueprint Node:

Input Parameters

Parameter Name Data Type Description
Exec Exec Triggers the file deletion logic.
Replay Name String **Required:** The name of the target replay file to delete.

Output Parameters

Parameter Name Data Type Description
Exec Exec Triggers once the deletion attempt is complete.
Return Value Boolean **Success Flag:** Returns True if deleted; False if the file was not found or deletion failed.

Get Replay Total Duration (Get Replay Total Duration)

Called during playback to return the total length of the current replay. Useful for setting the maximum value of a progress bar.

Blueprint Node:

Output Parameters

Parameter Name Data Type Description
Return Value Float **Value:** The total duration of the replay (in seconds). Usually returns 0.0 if not in a replay.

Get Current Replay Time (Get Current Replay Time)

Used to track replay progress in real-time. It can be used with Tick or a timer to synchronize and update the progress bar and time text on the UI.

Blueprint Node:

Input Parameters (Input)

Parameter Name Data Type Description
Exec Exec Triggers the instruction to get the current playback progress.

Output Parameters (Output)

Parameter Name Data Type Description
Exec Exec Triggered after the instruction has been executed.
Return Value Float **Return Value:** The current position of the replay in seconds (e.g., 15.5 seconds).

Seek To Time (Seek To Time)

Allows players to jump directly to a specific timestamp in the replay via a progress bar or by entering a value. Commonly used for quickly locating highlight moments.

Blueprint Node:

Input Parameters (Input)

Parameter Name Data Type Description
Exec Exec Triggers the seek instruction.
Time in Seconds Float **Target Time:** The time point you want to jump to (in seconds). Ensure the value is between 0 and the total duration.

Output Parameters (Output)

Parameter Name Data Type Description
Exec Exec Triggered after the seek request is issued.
Return Value Boolean **Return Value:** Whether the seek operation was successfully initiated.

Set Replay Playback Speed (Set Replay Playback Speed)

Adjusts the playback rate. For example, 2.0 is double speed, while 0.5 is slow motion.

Blueprint Node:

Input Parameters

Parameter Name Data Type Description
Playback Speed Float **Rate:** Default is 1.0.
  • > 1.0: Fast forward
  • < 1.0: Slow motion
  • 0.0: Pause (implementation dependent)

Pause Replay (Pause Replay)

Called during the replay process to freeze the current replay frame and progress. Often used to implement pause functionality in a player interface.

Blueprint Node:

Input Parameters (Input)

Parameter Name Data Type Description
Exec Exec Triggers the logic command to pause the replay.

Output Parameters (Output)

Parameter Name Data Type Description
Exec Exec Triggered after the pause command has been executed.
Return Value Boolean **Return Value:** Indicates if the pause was successful. Returns False if the world is not currently in replay mode.

Resume Replay (Resume Replay)

Used to restart playback for a replay that is currently paused, continuing along the recorded timeline.

Blueprint Node:

Input Parameters (Input)

Parameter Name Data Type Description
Exec Exec Triggers the logic command to resume playback.

Output Parameters (Output)

Parameter Name Data Type Description
Exec Exec Triggered after the resume command has been executed.
Return Value Boolean **Return Value:** Indicates whether playback was successfully resumed.
